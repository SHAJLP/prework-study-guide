HTML

  • The head element contains information about the website.
  • The body element represents the visible content shown to the user.
  • HTML HyperText Markup Language, basic building block of websites
  • The head element contains information about the website.
  • The body element represents the visible content shown to the user.
  • The body element is also known as the parent element, as they are other elements nested inside
  • HTML code starts with a declaration, this tells the browser what document type it should expect when opened <DOCTYPE html>
  • This is followed by &l;thtml> which represents the start of your HTML document
  • Within above tag is an attribute of lang="en", which tells it which language to use.
  • The <head> element is next and contains Metadata for the web application (Metadata contains information about accessibility, search engines and performance )
  • Inside the <head> element we can use metadata tags like <title>
  • The <title> element is extremely important for search engine optimisation
  • element defines the base URL for a webpage
  • The <style> element defines the CSS styles associated with a website
  • The <link> element connects an external resource to the HTML document.
  • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport.
  • Use <header> so this appears at the top of page and usually contains navigation menu or introductory menu.
  • You can use additional content such as an image by adding <h1>under <header> and adding as many as required
  • An image can be added by using <img>within your header/s
  • There is <main> element which will start the of text followed by </main> to finish
  • <footer> element is at the bottom of the page and usually contains information about copyrights, contact or whatever you choose
  • You can add <sections> to HTML page within the <body> to make it easier to organise different subject areas, making it easier to find the information you need

CSS

  • CSS "cascading style sheets" add styling,fonts spacing and colours to website
  • CSS has three different ways to style a webpage, inline CSS, external CSS, internal CSS
    • * Internal CSS is used to define a style for a single HTML within a <style> * Inline CSS may be used to apply a unique style for a single element * External CSS is used to change the look of an entire website, multiple webpages can use the same link
  • You can add margins, this indicates how much space we want around the outside of our element.
  • Margins are not visible on page to the user
  • A Margin indicates how much space we want around the outside of an element
  • The padding adds space around the content inside your element
  • You can set a position for your information by using 'text-align'

Git

  • git status: checks what branch we are currently on
  • git checkout

    Git

    • Version control system that lets you manage and track each version of a project
    • Git commands always start with "git"
    • With Git commands you can create, initiate a repo
    • Git allows you to track changes on your own hardware
    • Git based repository services allows you to collaborate on projects
    • git status: checks what branch we are currently on
    • git checkout -b branch-name: creates a new branch and switches to it
    • CLI-Command Line Interface, allows you to input text commands to execute operating systems
    • Terminal is the application that allows us to interact with the 'shell'.
    • A package manager makes it easy to install, upgrade, configure and remove packages.
    • CLI, uses commands to navigate system and to open files.
    • SSH key is an access credential for the secure shell network.
    • Generating SSH keys may provide an opportunity to debug and solve the problem
    • Without a SSH key you will have to enter a password every time you push your code to Github
    • setup a Github account by visiting Github website and follow instructions to set up a new account.
    • You must check if you have existing SSH keys on your computer, read "checking for existing SSH keys", from Github website
    • Github uses the email address set in your local Git configuration to associate commits pushed from the command line with your Github account
    • Git is a version control system that allows you to mange and keep track of each version of your project
    • Git can be used for managing and tracking changes locally on your computer
    • Github is a cloud based service which hosts your repositories and where project codes will live and Git is a version control software
    • Developers use repositories as folders to store, files that contain code files that contain information like README.files,the version history for all the files in the repositories
    • Repositories can be owned by one developer or shared by a team/organisation.
    • Github offers a task management tool called Github issues, its an easier way to keep track of key tasks and progress.
    • Github issues can help to track, organize, amend, and enhance tasks.
    • Download Visual Studio Guide, to write, view and edit code
    • When working on a collaboration, you must setup a feature branch from the main branch. this ensures you are not making any changes directly to the main page and allows you to make changes before they are committed
    • You must add a commit message that describes the changes you have made to the code
    • The changes are then pushed to the feature branch in your Github repo
    • To merge changes to the main branch you must open a Pull Request
    • This allows other developers/collaborators to view changes you have made
    • Once the changes have been approved, it can be merged on to the main branch
    • Always check your local branch is in sync with the main branch
